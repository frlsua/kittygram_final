# Kittygram CI/CD
Веб-платформа для публикации фотографий и информации о своих питомцев. Позволяет узнавать о котах других владельцев.

## Цель проекта
Настроить запуск проекта Kittygram в контейнерах и CI/CD с помощью GitHub Actions

## Используемый стек и технологии
| Слой | Стек |
| - | - |
| Frontend | React, Java Script |
| Backend | Django, Gunicorn |
| Data Base | PostgreSQL |
| Autentification | JWT |
| Контейнеризация | Docker, DockerHub |
| CI/CD | GutHub Actions |

## Сборка проекта в контейнере
```
docker compose up --build
```

## Переменные окружения
Все настройки задаются через .env (см. .env.example):

| Переменная | Описание |
| - | - |
| POSTGRES_DB | При запуске образа PostgreSQL используется для автоматического создания базы данных с заданным именем |
| POSTGRES_USER | Определения имени суперпользователя базы данных |
| POSTGRES_PASSWORD | Пароль для суперпользователя |
| DB_NAME | Задает имя базы данных, создаваемой автоматически при первом запуске контейнера с СУБД|
| SECRET_KEY | Уникальная случайная строка, используемая для обеспечения безопасности веб-приложения (криптографической подписи) |
| ALLOWED_HOSTS | Список строк с именами доменов или IP-адресов, которые разрешено обслуживать текущему приложению |
| DB_HOST | Хост базы данных |
| DB_PORT | Обслуживаемый базой данных порт |
